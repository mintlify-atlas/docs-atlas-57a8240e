## Overview

The Batch Jobs module provides data processing pipelines that transform data through the medallion architecture (Bronze → Silver → Gold). It orchestrates complex transformations using Apache Spark and writes to multiple storage systems including ClickHouse, Neo4j, and Pinecone.

## Architecture

<CardGroup cols={3}>
  <Card title="Bronze → Silver" icon="gem">
    Deduplication and timestamp-based upserting
  </Card>
  
  <Card title="Silver → Silver" icon="database">
    Extract and transform to ClickHouse analytics database
  </Card>
  
  <Card title="Silver → Gold" icon="share-nodes">

  </Card>
</CardGroup>

## Pipeline Components

### Bronze to Silver Pipeline

<ParamField path="run_dedup_timestamp" type="function">
  Deduplicates records based on timestamp from Bronze to Silver layer in Delta Lake.

  **Location**: `src/batch_jobs/pipelines/bronze_silver/minio_to_minio.py:19`

  **Process Flow**:
  1. Loads configuration and initializes Spark session
  2. Gets last processed version from Redis
  3. Reads Change Data Feed (CDF) from Bronze Delta tables
  4. Performs upsert based on timestamp to keep latest records
  5. Writes deduplicated data to Silver layer
  6. Updates version in Redis

  **Data Types Processed**: movie, tv_series, person

  **Key Operations**:
  - Change Data Feed reading for incremental processing
  - Timestamp-based deduplication
  - Batch version tracking in Redis
</ParamField>

### Silver to Silver Pipeline

<ParamField path="write_minio_to_clickhouse" type="function">
  Transforms and writes data from Delta Lake (Silver) to ClickHouse tables.

  **Location**: `src/batch_jobs/pipelines/silver_silver/minio_to_clickhouse.py:80`

  **Process Flow**:
  1. Initializes ClickHouse tables
  2. Reads Silver layer Delta tables filtered by batch version
  3. Transforms data for each ClickHouse table using transform functions
  4. Writes to ClickHouse using native protocol
  5. Processes multiple tables per data type

  **Tables Created**:
  - **movie**: movie, movie_cast, movie_crew
  - **person**: person
  - **tv_series**: tv_series, tv_series_cast, tv_series_crew

  **Features**:
  - Native ClickHouse protocol for high performance
  - Batch version tracking for idempotency
  - Schema transformation and denormalization
</ParamField>

### Silver to Gold Pipelines

#### Neo4j Knowledge Graph

<ParamField path="write_clickhouse_to_neo4j" type="function">
  Builds knowledge graph in Neo4j from ClickHouse data.

  **Location**: `src/batch_jobs/pipelines/silver_gold/clickhouse_to_neo4j.py:181`

  **Process Flow**:
  1. Processes nodes (Movie, Person, TV_Series)
  2. Creates relationships (ACTED_IN, WORKS_ON)
  3. Handles relationship adds and removes from diff columns
  4. Uses batch version for incremental updates

  **Node Labels**:
  - **Movie**: movie_id, title, overview, popularity, etc.
  - **Person**: person_id, name, biography, birthday, etc.
  - **TV_Series**: tv_series_id, overview, first_air_date, etc.

  **Relationships**:
  - **ACTED_IN**: Person → Movie/TV_Series (with cast details)
  - **WORKS_ON**: Person → Movie/TV_Series (with crew details)

  **Features**:
  - Constraint-based node creation
  - Relationship properties (character, job, department)
  - Handles adds/removes from change tracking
</ParamField>

#### Pinecone Vector Index

<ParamField path="write_clickhouse_to_pinecone" type="function">
  Creates vector embeddings and writes to Pinecone for semantic search.

  **Location**: `src/batch_jobs/pipelines/silver_gold/clickhouse_to_pinecone.py:30`

  **Process Flow**:
  1. Reads movie and tv_series data from ClickHouse
  2. Prepares vector schema (combines overview + tagline)
  3. Generates embeddings using e5-large-v2 model
  4. Writes vectors to Pinecone with metadata

  **Transform Map**:
  ```python
  {
      "movie": {
          "id_col": "movie_id",
          "vector_prepare_cols": ["overview", "tagline"],
          "vector_col_name": "document",
          "metadata": "{}"
      },
      "tv_series": {
          "id_col": "tv_series_id",
          "vector_prepare_cols": ["overview", "tagline"],
          "vector_col_name": "document",
          "metadata": "{}"
      }
  }
  ```

  **Embedding Model**: intfloat/e5-large-v2

  **Namespaces**: Separate namespace per data type
</ParamField>

### Reading Delta Tables

```python
from batch_jobs.io.readers.delta_minio_reader import DeltaMinioReader
from pyspark.sql import SparkSession

spark = SparkSession.builder.getOrCreate()
reader = DeltaMinioReader(spark)

# Read full table
df = reader.read_table("s3a://silver/movie/valid")

# Read with filters
df = reader.read_table_with_filters(
    target_path="s3a://silver/movie/valid",
    filters={"batch_version": 42}
)

# Read Change Data Feed
df = reader.read_table_cdf(
    target_path="s3a://bronze/movie/valid",
    start_version=10,
    end_version=20
)

# CDF shows what changed
df.filter(df._change_type == "insert").show()
```

### Writing to Neo4j

```python
from batch_jobs.io.writers.neo4j_writer import Neo4jWriter
from pyspark.sql import SparkSession

spark = SparkSession.builder.getOrCreate()
writer = Neo4jWriter(spark)

# Write nodes
movie_df = spark.createDataFrame([
    {"movie_id": 1, "title": "The Matrix", "year": 1999}
])

writer.write_node_constraint(
    df=movie_df,
    label="Movie",
    keys=["movie_id"]
)

# Write relationships
acted_in_df = spark.createDataFrame([
    {"person_id": 100, "movie_id": 1, "character": "Neo"}
])

writer.write_relationship(
    df=acted_in_df,
    repartition_cols=["movie_id"],
    relationship="ACTED_IN",
    source_label="Person",
    source_keys=["person_id"],
    source_properties=[],
    target_label="Movie",
    target_keys=["movie_id"],
    target_properties=[],
    relationship_properties=["character"]
)
```

### Writing to Pinecone

```python
from batch_jobs.io.writers.pinecone_writer import PineconeWriter
from pyspark.sql import SparkSession

spark = SparkSession.builder.getOrCreate()
writer = PineconeWriter(spark)

# DataFrame must have: id, values, namespace, metadata
vector_df = spark.createDataFrame([
    {
        "id": "movie_1",
        "values": [0.1, 0.2, ..., 0.5],  # 1024-dim vector
        "namespace": "movies",
        "metadata": '{"title": "The Matrix"}'
    }
])

writer.write_index(vector_df)
```

## Orchestration (Airflow)

The batch jobs can be orchestrated with Apache Airflow:

```python
# Example DAG structure (from batch_jobs/dags/batch_jobs.py)

@dag(schedule="*/10 * * * *", catchup=False)
def full_batch_jobs():
    
    @task()
    def job_dedup_timestamp():
        run_dedup_timestamp()
    
    @task()
    def job_extract_to_clickhouse():
        write_minio_to_clickhouse()
    
    @task()
    def job_write_to_neo4j():
        write_clickhouse_to_neo4j()
    
    @task()
    def job_write_to_pinecone():
        write_clickhouse_to_pinecone()
    
    # Pipeline dependencies
    (job_dedup_timestamp() >> 
     job_extract_to_clickhouse() >> 
     job_write_to_neo4j() >> 
     job_write_to_pinecone())
```

## Data Flow

<Steps>
  <Step title="Bronze → Silver">
    Deduplicate records by timestamp using Change Data Feed
  </Step>
  
  <Step title="Silver → ClickHouse">
    Transform and denormalize for analytical queries
  </Step>
  
  <Step title="ClickHouse → Neo4j">
    Build knowledge graph with entities and relationships
  </Step>
  
  <Step title="ClickHouse → Pinecone">
    Generate embeddings for semantic search
  </Step>
</Steps>

## Best Practices

<Tip>
**Version Tracking**: Always use Redis to track batch versions for idempotent processing

**Incremental Processing**: Leverage Change Data Feed to process only changed data

**Error Recovery**: Batch version tracking allows reprocessing from last successful state

**Parallelism**: Configure Spark partitions based on data volume and cluster size

**Monitoring**: Track batch versions, record counts, and processing times
</Tip>

## Configuration

Environment variables and settings are loaded from YAML configuration files similar to other modules, with additional settings for:

- **Spark**: Application names, master URL, executor configuration
- **ClickHouse**: JDBC URL, credentials, batch sizes
- **Neo4j**: Connection URI, credentials, batch sizes
- **Pinecone**: API key, index name, namespaces
- **Redis**: Host, port, key prefixes for version tracking
